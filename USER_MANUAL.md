## 1. การเริ่มต้นใช้งาน

### ความต้องการของระบบ

| รายการ          | ขั้นต่ำ                  |
|-----------------|--------------------------|
| Docker Desktop  | version 24+              |
| RAM             | 4 GB                     |
| Disk            | 2 GB                     |
| Internet        | ต้องการ (ดึงราคา API)     |

### ขั้นตอนติดตั้ง

**ขั้นตอนที่ 1: ตั้งค่า API Key**

เปิดไฟล์ `.env` ที่ root ของ project แล้วแก้ไข:

```env
ALPHA_VANTAGE_API_KEY_GOLD=ใส่ key ของคุณที่นี่
ALPHA_VANTAGE_API_KEY_OIL=ใส่ key ที่สองที่นี่
```

รับ API Key ฟรีได้ที่ https://www.alphavantage.co/support/#api-key  
(สมัครใหม่จะได้ key ทันที ไม่ต้องรอ)

**ขั้นตอนที่ 2: รันระบบ**

```bash
docker-compose up --build
```

**ขั้นตอนที่ 3: รอระบบโหลด**

ครั้งแรกระบบจะโหลดข้อมูลย้อนหลัง 20 ปี (~124,000 rows) ใช้เวลา **2–5 นาที**  
เมื่อเห็น log ว่า `Uvicorn running on http://0.0.0.0:8000` แสดงว่าพร้อมใช้งาน

**ขั้นตอนที่ 4: เปิดเว็บ**

เปิด browser ไปที่ **http://localhost:3000**

## 3. หน้า Prediction Result

แสดงผลการคาดการณ์ของ AI และประวัติการซื้อขาย

### คำแนะนำ AI

| ฟิลด์          | ความหมาย                                               |
|----------------|-------------------------------------------------------|
| Action         | **BUY** / **SELL** / **HOLD** — ทิศทางที่แนะนำ       |
| Confidence     | ความมั่นใจของโมเดล (0–100%) ยิ่งสูงยิ่งมั่นใจ        |
| Entry Price    | ราคาทองคำ ณ เวลาที่คาดการณ์                           |
| Take Profit    | ราคาเป้าหมายกำไร                                       |
| Stop Loss      | ราคาตัดขาดทุน                                          |
| Risk %         | สัดส่วน capital ที่โมเดลแนะนำเสี่ยงต่อ trade นี้     |

**ตัวอย่างการอ่านผล:**
```
Action:       BUY
Confidence:   82.3%
Entry:        5,173.50
Take Profit:  5,250.00  (+1.47%)
Stop Loss:    5,130.00  (−0.84%)
Risk:         1.2%
```
→ โมเดลแนะนำซื้อที่ 5,173.50 เป้า 5,250.00 ตัดขาดทุนที่ 5,130.00

### ปุ่ม Predict Now

กดเพื่อขอการคาดการณ์ทันที (ไม่รอ scheduler)  
ผลที่ได้จะ**ไม่**เปิด trade simulation — ใช้เพื่อดูข้อมูลเท่านั้น

### ตาราง Trade History

แสดงประวัติ trade simulation ที่ระบบเปิด/ปิดอัตโนมัติ:

| คอลัมน์    | ความหมาย                           |
|------------|-------------------------------------|
| Status     | OPEN / WIN / LOSS                   |
| Direction  | BUY หรือ SELL                       |
| Entry      | ราคาที่เปิด trade                   |
| Exit       | ราคาที่ปิด trade (TP หรือ SL)       |
| Profit     | กำไร/ขาดทุน ($)                    |

## 4. หน้า Model Performance

แสดงสถิติประสิทธิภาพของโมเดล AI

### ตัวชี้วัดหลัก

| ตัวชี้วัด      | ความหมาย                                                   | เกณฑ์ดี     |
|---------------|-----------------------------------------------------------|-------------|

กราฟเส้นแสดงการเติบโตของ equity ตามเวลา  
เส้นที่ขึ้นสม่ำเสมอโดยไม่มี drawdown ลึก = โมเดลมีเสถียรภาพดี

## 6. การตีความผลการคาดการณ์

### ระดับความน่าเชื่อถือ

| Confidence    | ระดับ       | คำอธิบาย                                    |
|---------------|-------------|---------------------------------------------|
| > 80%         | สูงมาก      | โมเดลมั่นใจสูง สัญญาณชัดเจน                |
| 60–80%        | สูง         | สัญญาณดี ควรพิจารณาประกอบปัจจัยอื่น         |
| 40–60%        | ปานกลาง    | สัญญาณไม่ชัด ควรระวัง                       |
| < 40%         | ต่ำ         | ไม่แนะนำให้ใช้เป็นสัญญาณหลัก               |

### ข้อควรระวัง

- โมเดลนี้ฝึกด้วยข้อมูล **2005–2022** ซึ่งส่วนใหญ่เป็นช่วง **ตลาดขาขึ้นระยะยาว**
- ดังนั้นโมเดลมักแนะนำ **BUY** บ่อยกว่า SELL — เป็นพฤติกรรมปกติของโมเดล
- ในช่วงตลาดขาลงหรือ sideways ประสิทธิภาพอาจต่ำกว่าในช่วงขาขึ้น
- ไม่ควรใช้สัญญาณนี้เพียงอย่างเดียวในการลงทุนจริง

## 7. ระบบ Trade Simulation

ระบบจำลองการซื้อขายโดยอัตโนมัติตามสัญญาณของโมเดล

### วงจรของ Trade

```
Scheduler :05 → predict → เปิด trade (ถ้า BUY/SELL confidence > 10%)
                              ↓
Scheduler :02/:05 → check_open_trades()
                              ↓
           scan ทุก bar ตั้งแต่ open_time
                              ↓
         bar ใดก็ตามที่ High ≥ TP → ปิด WIN
         bar ใดก็ตามที่ Low  ≤ SL → ปิด LOSS
```

### กฎการเปิด Trade

- เปิดได้แค่ **1 trade ต่อครั้ง** (ไม่มี multiple open positions)
- เปิดเฉพาะเมื่อ action = BUY หรือ SELL และ confidence > 10%
- ขนาด lots คำนวณจาก: `lots = (10,000 × risk%) / (entry × sl% × 100)`

### กฎการปิด Trade (v23 Fix)

ระบบสแกน **ทุก bar** ตั้งแต่เวลาเปิด trade ตามลำดับเวลา:
1. ตรวจ **TP ก่อน** (high ≥ tp_price สำหรับ BUY, low ≤ tp_price สำหรับ SELL)
2. ถ้าไม่ hit TP ตรวจ **SL** (low ≤ sl_price สำหรับ BUY, high ≥ sl_price สำหรับ SELL)
3. ปิดที่ **bar แรก** ที่พบ — ไม่ว่าจะผ่านมากี่ชั่วโมงแล้ว

> **v22 และก่อนหน้า:** ตรวจแค่ bar เดียว (latest) ทำให้ trade ค้างอยู่นานเกินควร

### การอ่านผล Trade

**Trade ที่ปิดแล้ว:**
- `WIN` — ราคาถึง Take Profit, กำไร = (TP − Entry) × lots × 100
- `LOSS` — ราคาถึง Stop Loss, ขาดทุน = (Entry − SL) × lots × 100

**Trade ที่ยังเปิดอยู่:**
- `OPEN` — ยังไม่ hit TP หรือ SL, กำไร/ขาดทุนยังไม่ถูก lock

## 8. การ Retrain โมเดล

### Retrain อัตโนมัติ

ระบบ retrain โมเดลอัตโนมัติทุกวันที่ 1 ของเดือน เวลา 02:00 UTC  
ใช้ข้อมูลราคาทั้งหมดที่มีใน database ณ เวลานั้น

### Retrain ด้วยตนเอง

**วิธีที่ 1: ผ่านหน้าเว็บ**
1. ไปหน้า **Model Performance**
2. กดปุ่ม **"Retrain Model"**
3. รอจนกว่า status จะเปลี่ยนเป็น SUCCESS

**วิธีที่ 2: ผ่าน API**
```bash
curl -X POST http://localhost:8000/api/model/train
```

**วิธีที่ 3: ผ่าน Swagger UI**
1. เปิด http://localhost:8000/docs
2. ไปที่ `POST /api/model/train`
3. กด **Try it out** → **Execute**

### ระยะเวลา Retrain

ขึ้นอยู่กับ hardware และขนาด dataset:
- CPU เท่านั้น: ~20–40 นาที
- GPU: ~5–10 นาที

ระหว่าง retrain ระบบยังทำงานปกติ (ใช้โมเดลเดิมก่อน)  
เมื่อ retrain เสร็จ โมเดลใหม่จะถูกโหลดโดยอัตโนมัติ

## 9. คำถามที่พบบ่อย

**Q: ระบบไม่แสดงราคาล่าสุด ทำไม?**  
A: ตรวจสอบ API Key ใน `.env` และตรวจ internet connection  
ดู log ด้วย `docker-compose logs backend | grep "fetch"`

**Q: Trade ที่เปิดไว้ทำไมไม่ปิด?**  
A: v23 แก้บัคนี้แล้ว Trade จะปิดอัตโนมัติเมื่อ TP หรือ SL ถูก hit  
ระบบจะตรวจที่ :02 (หลังดึงราคา) และ :05 (หลัง predict) ทุกชั่วโมง

**Q: โมเดลแนะนำ BUY ตลอด ปกติไหม?**  
A: ปกติครับ โมเดลฝึกด้วยข้อมูล 2005–2022 ซึ่งทองเป็นขาขึ้นระยะยาว  
ทำให้ Policy เรียนรู้ว่า BUY ให้ผลดีในระยะยาว

**Q: Confidence 100% แปลว่าอะไร?**  
A: แปลว่า raw action[0] จากโมเดลมีค่า ±∞ หลัง tanh → ±1.0  
ในทางปฏิบัติหมายถึงโมเดลมั่นใจสูงมาก แต่ไม่ได้แปลว่าถูกเสมอ

**Q: ราคาอัพเดทบ่อยแค่ไหน?**  
A: ราคาดึงจาก AlphaVantage ทุก 1 ชั่วโมงที่ :00 UTC  
Chart ในหน้า Dashboard refresh อัตโนมัติทุก 2 นาที

**Q: จะดู log ของระบบได้อย่างไร?**  
A: รันคำสั่ง:
```bash
# ดู log ทั้งหมด
docker-compose logs -f

# ดูเฉพาะ backend
docker-compose logs -f backend

# ดูเฉพาะ trade check
docker-compose logs backend | grep "Trade #"
```

**Q: ข้อมูลเก็บอยู่ที่ไหน?**  
A: ใน SQLite database: `backend/database/gold_trading.db`  
สามารถเปิดด้วย DB Browser for SQLite หรือ DBeaver

**Q: หยุดและรันใหม่แล้วข้อมูลหายไหม?**  
A: ไม่หาย ข้อมูลเก็บใน `gold_trading.db` ซึ่งเป็น volume ของ Docker  
ข้อมูลจะหายก็ต่อเมื่อลบ volume ด้วย `docker-compose down -v`

## Appendix: การอ่าน Indicator

### RSI (Relative Strength Index)
- คำนวณจากอัตราส่วนราคาขึ้น vs ราคาลงใน 14 ชั่วโมงย้อนหลัง
- ค่า > 70 → Overbought (อาจปรับฐาน)
- ค่า < 30 → Oversold (อาจเด้งขึ้น)
- ค่า 50 → Neutral

### MACD
- MACD Line = EMA12 − EMA26
- Signal Line = EMA9 ของ MACD
- MACD > Signal + Histogram บวก → แนวโน้มขาขึ้น
- MACD < Signal + Histogram ลบ → แนวโน้มขาลง

### Bollinger Bands
- กลาง = SMA20, บน/ล่าง = ±2 Standard Deviation
- ราคาชนแถบบน → Overbought
- ราคาชนแถบล่าง → Oversold
- แถบแคบ (Squeeze) → ความผันผวนต่ำ อาจระเบิดเร็วๆ นี้

### EMA Cross
- EMA50 ตัด EMA200 ขึ้น (Golden Cross) → สัญญาณขาขึ้น
- EMA50 ตัด EMA200 ลง (Death Cross) → สัญญาณขาลง
